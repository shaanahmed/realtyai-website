1. The Website: Google Sites (Free + Custom Domain)

Since you do not have a web app yet, you only need a static website that explains the product, builds trust, and gives customers a place to read support information.

Recommended tool: Google Sites

Why Google Sites is the best $0 option:
- It is free.
- It works well on mobile.
- It is simple drag-and-drop.
- You can connect your custom Namecheap domain for free.
- It does not look cluttered with builder branding like some other free tools.
- It is enough for a desktop app landing page, support page, privacy page, and help center links.

What the website should include:
- Home page / landing page
- Features page
- How It Works page
- How to Use RealtyAI page
- Security & Privacy page
- Contact / Support page
- Download or Buy page
- Help / Manuals page

How to connect it:
1. Go to sites.google.com.
2. Build the website pages.
3. Open Settings in Google Sites.
4. Go to Custom Domains.
5. Enter realtyaihq.com.
6. Google will give you a CNAME record and a TXT record.
7. Open Namecheap Advanced DNS.
8. Add those records there.
9. Wait for the domain to verify and propagate.

1A. App Updates Hosting: GitHub Releases (Free)

Recommended tool: GitHub Releases in a separate public repo used only for release assets.

Why this is the best fit:
- It is free.
- It gives the app one stable internet location for update files.
- It does not require a separate server.
- It keeps your source code private if you use a separate public updates repo.
- It works well with the RealtyAI desktop updater flow.

Best setup:
- Website and support pages live on realtyaihq.com.
- Desktop update files live in a separate public GitHub repo such as realtyai-updates.
- The RealtyAI app checks GitHub Releases for latest.json and installer assets.

What the app needs online:
- latest.json
- the Windows installer .exe
- the installer signature .sig

Important separation:
- Your website is for trust, product information, support, and help content.
- Your GitHub Releases repo is only for app update delivery.
- Do not put your app source code in the public updates repo.

2. WHAT REALTYAI IS
==================================================

RealtyAI is a local-first desktop app for real estate professionals. It helps agents and small real estate teams create listing content, analyze market activity, prepare negotiations, review contract language, manage listing records, and export polished PDFs from one desktop workspace.

It is designed for people who want speed, privacy, simplicity, and practical business help instead of another bloated CRM or web subscription.

Short brand description:
RealtyAI is a private, desktop-based real estate productivity app that helps agents create better listing materials, understand the market faster, and make smarter deal decisions without depending on a heavy web platform.

Short one-line positioning:
RealtyAI helps real estate agents turn listing data into faster marketing, clearer decisions, and more professional client output from one local desktop app.

Simple elevator pitch:
RealtyAI is the assistant a solo agent or small team wishes they had every day. It helps write listing copy, organize listings, review market data, support negotiation prep, flag risky contract wording, and keep work moving fast without forcing the user into a complicated enterprise system.

5. WHAT THE APP DOES
==================================================

Core app capabilities:

1. Listing Generator
- User enters the property basics once.
- RealtyAI generates ready-to-use marketing copy.
- The copy can be refined with extra notes and local details.
- This helps the agent go from blank page to publish-ready draft much faster.

2. Dashboard
- Shows saved listings in one place.
- Lets the user search, filter, edit, restore, delete, and export listings.
- Tracks listing history and recent changes.
- Helps the user stay organized without digging through scattered files.

3. AI Studio
- Shows market snapshot information in a simpler decision-oriented view.
- Supports batch listing generation for multiple properties.
- Provides negotiation prep assistance.
- Flags risky contract language for review.
- Gives the user a more advanced workspace without forcing them into a messy, confusing experience.

4. Portfolio Management
- Lets the user review groups of listings together.
- Supports filtering by city and portfolio views.
- Helps the user compare values, trends, and bulk changes.
- Useful for agents managing many listings or repeating workflow across areas.

5. PDF Export
- Turns listing information into professional PDF output.
- Useful for client presentations, internal review, and branded materials.

6. Settings and AI Setup
- Lets the user add API keys for supported AI providers.
- Gives guidance on local AI setup.
- Helps the user choose between local AI and cloud AI.
- Includes update and maintenance actions in the desktop app.

7. License and Access Control
- Supports licensing flow for paid distribution.
- Gives a path for desktop commercial sales.

7. HOW TO USE THE APP
==================================================

Recommended basic workflow for a new user:

Step 1: Install the app on Windows.

Step 2: Open Settings.
- Add API key if using cloud AI.
- Or set up local AI if preferred.
- Check model recommendation.
- Activate the license if required.

Step 3: Start with Listing Generator.
- Enter address, price, beds, baths, square feet, and extra notes.
- Generate listing copy.
- Review and personalize one local detail before publishing.

Step 4: Use Dashboard.
- Save listings.
- Search and filter them later.
- Edit details when needed.
- Export polished PDFs.

Step 5: Use AI Studio for advanced help.
- Review market snapshot.
- Run batch listing generation.
- Use negotiation prep before client or buyer conversations.
- Paste contract text to flag areas that may need closer attention.

Step 6: Use Portfolio view when managing many listings.
- Compare activity across listings.
- Apply smarter bulk review and management.

Best practice for non-technical users:
- Start with one listing.
- Learn the basic generate, review, save, export flow first.
- Move to AI Studio only after getting comfortable.

8. WHAT USERS SHOULD KEEP IN MIND
==================================================

Important expectations and user guidance:

1. This is a desktop app, not a web app.
- It is meant to run locally on the user's computer.

2. Some features may depend on AI configuration.

- Listing copy should be checked for accuracy.

5. Internet may still be required for some tasks.
- Cloud AI needs internet.
- License activation may need internet.
- Update checks need internet.

6. Privacy and security still matter.
- Users should protect their machine.

22. SHORT FAQ COPY
==================================================

Q: Is RealtyAI a web app?
A: No. RealtyAI is a Windows desktop app.

Q: Do I need internet?
A: Some features can work with a local AI setup, but cloud AI, updates, and some licensing actions may require internet access.

Q: Do I need an API key?
A: If you use cloud AI providers, yes. If you use a supported local AI setup, your workflow may not require a cloud key for those tasks.

Q: Is my data stored online?
A: RealtyAI is designed as a local-first desktop workflow. Cloud AI requests only go to the provider the user chooses to connect.

Q: Can I use RealtyAI if I am not technical?
A: Yes. RealtyAI is designed to keep the main workflow simple so users can start with listing generation and learn advanced tools gradually.

Q: Is the contract tool legal advice?
A: No. It is a review support tool that helps highlight areas for closer attention.
